# Voice & Tone

## Attributes

| Attribute | Meaning |
| --- | --- |
| **Professional** | Credible platform for coaches, parents, and organizers |
| **Athlete-first** | Players and their verified performance come first |
| **Trustworthy** | Only publish verified data; be clear about sample sizes |
| **Data-driven** | Lead with stats and evidence, not hype |
| **Modern** | Clean, current language — not dated sports clichés |
| **Approachable** | Plain language; avoid gatekeeping jargon |

## Writing rules

- Use **verified** when data comes from validated games.
- State sample sizes: “based on 16 games” not “all season.”
- Prefer specific numbers over vague praise.
- Age groups: **U13**, **U16**, **U19** (official).
- Avoid: “elite,” “game-changer,” “next big thing,” “unlock your potential.”

## UI copy patterns

| Context | Pattern |
| --- | --- |
| Empty state | What’s missing + what happens when data exists |
| Errors | What failed + what the user can do |
| Rankings | “National rank” with age group and gender |
| Stats footer | “Stats are based on the selected sample of N games.” |

## Product name

Always **Peach Basket** (two words) in user-facing copy.
